# docs/

All project documentation. Pick the folder that matches what you need.

## Folders

| Folder | Contents |
|---|---|
| `sessions/` | Session reports from each Claude Code working session — what was built, decisions made, bugs fixed |
| `audits/` | Tool logic audits, psychology audits, design consistency reports, module-by-module audit reports |
| `architecture/` | System design, schema design, naming conventions |
| `database/` | SQL migration files, schema documentation |
| `deployment/` | Railway deploy guides, LearnWorlds setup, roadmap, deployment checklist |
| `fast-track-specs/` | Original Fast Track program specs — tool criteria, writing guides, brand guidelines, FT fundamentals |
| `implementation/` | Implementation summaries for major features (guru suite, Supabase integration, tool reorganization) |
| `learnworlds-api/` | LearnWorlds API docs — authentication, SSO, webhooks |
| `prompts/` | Reusable prompts for Claude (Supabase connection, guru modal build) |
| `source-materials/` | Reference PDFs and docs from the Fast Track program (sprints 10–27) |
| `templates/` | Tool build template, build checklist, build prompt, design checklist |
| `cognitive-load-research/` | Research and specs for the cognitive load system (case studies, info boxes) |

## Session reports

Named `SESSION-REPORT-YYYY-MM-DD.md`. Read these to understand what was built in any given session and why decisions were made. Useful before starting a new session.
